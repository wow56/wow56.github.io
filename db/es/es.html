# 简介

Elasticsearch 是一个 分布式 文档储存 中间件，它不会将信息储存为列数据行，而是储存已序列化为 JSON 文档的复杂数据结构。

ES 仅支持 JSON 格式数据。

# 数据格式

  • Index(索引) -> Database(数据库)
  • Type(类型) -> Table(表)
  • Document(文档) -> 一条数据
  • Fields(字段)

# 索引操作

创建名为 aa 的索引

PUT http://localhost:9200/aa

查询名为 aa 的索引

GET http://localhost:9200/aa

删除名为 aa 的索引

DELETE http://localhost:9200/aa

# 文档操作

ES 中的文档是基于索引的。

# 创建文档

POST http://localhost:9200/{index}/{type}/{id} body = JSON

  • 在 aa 索引上创建文档，如果索引 aa 不存在的话，默认会被创建出来
    • POST http://localhost:9200/aa/doc body = JSON
  • 指定 id 的值
    • POST http://localhost:9200/aa/doc/1001 body = JSON
    • 使用同一个 id 进行创建的话，version 会递增。

# 查看文档

list data: GET http://localhost:9200/aa/_search

find one: GET http://localhost:9200/{index}/{type}/{id}

返回结果示例：

{
    "_index": "aa",
    "_type": "doc",
    "_id": "1001",
    "_version": 4,
    "found": true,
    // source 中存放的就是文档内容
    "_source": {
        "name": "xxx",
        "description": "yyy"
    }
}

# 多条件查询

body 示例：

    "query": {
        "bool": {
            // must == and, should == or
            "must": [
                // 条件 1
                {
                    "match": {
                        // 一个 match 里面只能有一个条件（field）
                        "name": "aaa"
                    }
                },
                // 条件 2
                {
                    "match": {
                        "age": 18
                    }
                }
            ],
            "filter": {
                "range": {
                    "k": {
                        "gt": "ddd"
                    }
                }
            }
        }
    }
